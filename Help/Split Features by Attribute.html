Split Features by Attribute

Title Split Features by Attribute

Summary

This script is designed to take a field and use its unique values to explode a feature class into multiple feature classes within a selected workspace. Please keep in mind, that this will unload a lot of features into the desired workspace, so make sure to pick an empty workspace if you don't want to create clutter.


Usage

This tool can be used to separate a feature class into multiple feature classes based on a selected field. This tool can be used for everything from data management, tiling, to preparing separate layers for manipulation in CityEngine.


Syntax

ExplodeFeaturestoFCinFGDB (Input_Feature_Class, FGDB_Workspace, Explode_Field, Compact_Workspace)

Parameter Explanation Data Type
Input_Feature_Class Dialog Reference

This is the input feature class you want to be exploded into multiple feature classes. This feature class has a field of interest that you wish to split the feature class by.

Python Reference

This is the input feature class you want to be exploded into multiple feature classes. This feature class has a field of interest that you wish to split the feature class by. As part of the tool, a new field called "ExplodeID" is addeded to this feature class, and the field of interest is cast as a string to simplify and santize the inputs that the feature class will be split by.

 Feature Layer
FGDB_Workspace Dialog Reference

This is the output work space you want the new created feature classes to be placed. Make sure it is one you are ok with potentially many features being unloaded into (preferably empty).

___________________

Python Reference

Each new feature is put into the work space with the select analysis tool. It depends on the tools error handlying to place the new feature.

 Workspace
Explode_Field Dialog Reference

This represents the field from the input feature class that you wish to use to split the feature class into multiple features based on the unique values within the field. A text based "ExplodeID" field will be added to the feature class. Keep in mind that the Feature Class names will validated if the fields used do not make valid field names (objectIDs have letters inserted at the beginning of the name).

___________________

Python Reference

Each unique value within the field of interest will become part of the name of the new feature class. The feature class name is set to be "FC_"+Unique_Value, except it might be different because the output is put into a validate table name function.

Field
Compact_Workspace Dialog Reference

This boolean will compact the output workspace after the tool runs to allow for more efficient storage of the data after multiple feature classes are created.

___________________

Python Reference

This parameters leads to a simple boolean that will compact the output workspace data base after running.
