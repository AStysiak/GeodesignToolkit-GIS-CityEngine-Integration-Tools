Prepare Lot Associations

Title Prepare Lot Associations

Summary

This tool is designed to associate all records with one geometry for the purpose of creating standard geometry for CityEngine import and then used to drive rules as part of a geodesign methodology to create data driven lot based models. This tool is designed to take in polygon lot files that will be simplified into squares of a desired area, or area set by a field.


Usage

Use this tool to create an output feature class that has simplified geometry design for the purpose of CityEngine import.


Syntax

PrepareGeodataAssociations2 (Feature_Class, Output_Feature, {Area}, {Area_Field})

Parameter Explanation Data Type
Feature_Class Dialog Reference

The input feature classes whose features will be associated with a simplified version of the input feature class. The records will be preserved for the copy created, but the geometry will change.

___________________

Python Reference

Input feature class put into a copy features tool.

 Feature Layer
Output_Feature Dialog Reference

The resulting output feature class that has simplified geometry for importing into CityEngine. This feature class is a copy of the input feature class, except the geometry of each record has been replaced. Keep in mind that this output will be projected into Web Mercator Auxillary Sphere in order to make the feature CityEngine ready.

___________________

Python Reference

The copied feature is placed into an update cursor that creates a geometry centered on the mean center of the input file. The script is close to the same for lots or streets, as the script determines treatments based on feature type. If a geometry cannot be made, the record is deleted.

 Feature Class
Area (Optional) Dialog Reference

Is the area of the desired output lots that you wish your records to be associated with. Keep in mind that units are in terms of the feature classes current projection.

___________________

Python Reference

The tool essentially square roots the input to determine an appropriate length of each edge for the output lot. The tool uses the units of the current projection for node translations. Distances might not be entirely accurate due to changes in projection or based on various calculations.

 Double
Area_Field (Optional) Dialog Reference

If you want to reference a distance field (like Shape_Area or Shape_Length), that is in the units the same as the current projection, you can select a field here to use.
